Introduction
Background of the Study
0:35

Designing optimization algorithms has traditionally been a manual, expert-driven process. As the No Free Lunch Theorems establish, no single algorithm works best everywhere — so new algorithms must constantly be designed.

LLMs have changed this by enabling Automated Algorithm Design — frameworks like LLaMEA and Evolution of Heuristics use LLMs to iteratively generate and refine executable optimization code. However, these frameworks still have fundamental architectural limitations that motivated our work.

4
Introduction
Problem Statement
0:35

Building on those limitations, we identify three problems. First, underutilized populations — current frameworks rely on mutation only, with no recombination across candidates. Second, ineffective recombination — existing syntactic crossover simply concatenates code and produces invalid offspring. Third, no adaptive operator selection — static probabilities cannot respond to shifting exploration-exploitation needs during a run.

5
Introduction
Research Objectives
0:30

To address these, we defined three objectives. One, develop GA-LLaMEA with Discounted Thompson Sampling for adaptive operator selection. Two, implement Generative Crossover — multi-parent recombination via LLM reasoning. Three, validate on MA-BBOB using Elo ratings, A-O-C-C, behavioural profiling, and Code Evolution Graphs.

A-O-C-C ranges from zero to one, where higher means faster and more consistent convergence.

Introduction
Significance of the Study
0:25

Three contributions: First, we introduce Inspiration-Based Generative Crossover to the LLaMEA lineage. Second, we pioneer bandit-based adaptive control in LLM-driven evolutionary optimization via D-T-S. Third, we establish rigorous evaluation standards using MA-BBOB and A-O-C-C metrics as formalized by van Stein et al.

Literature
Theoretical Foundations
0:35

Our literature review addresses four questions. What does current AAD do? LLaMEA and EoH generate algorithms via LLM prompting, but remain mutation-centric. Why add adaptive selection? Static probabilities cannot track shifting operator effectiveness. Why D-T-S over UCB? D-T-S uses exponential discounting responsive to actual reward history, not a time-based clock. Why crossover? Mutation alone restricts search to local basins — no prior work introduces general-purpose crossover within LLaMEA for continuous optimization.

Research Gap — The Three Deficits
0:25

Three gaps: Gap one — mutation-centric evolution with no population recombination; solved by Generative Crossover. Gap two — syntactic crossover produces invalid offspring; solved by strategy-level semantic synthesis via LLM reasoning. Gap three — static operator scheduling; solved by D-T-S adaptive control with discounted posteriors.

Framework Architecture
0:35

On the left, Baseline LLaMEA: initialization, population management, then random selection among Refine, Random New, or Simplify. No recombination, purely random operator choice.

On the right, GA-LLaMEA adds two things: Generative Crossover as a fourth operator — multi-parent recombination via LLM reasoning — and Discounted Thompson Sampling replacing random selection with adaptive, reward-driven operator choice. These are our core contributions.
