Core Concepts

Skills & Agents

Skills = Abilities

# ~/.openclaw/skills/pr-review/SKILL.md

# PR Review Skill
Review pull requests with focus on:
- Code quality and patterns
- Security vulnerabilities
- Test coverage

## Steps
1. Fetch PR diff via `gh pr diff`
2. Analyze each changed file
3. Write structured assessment
4. Post review comments

Agents = Specialists

# openclaw.json → agents.list

{
  "id": "software-engineer",
  "name": "Ciel",
  "model": "gpt-5.3-codex",
  "identity": {
    "name": "Ciel",
    "theme": "Moonlit orchestrator",
    "emoji": "🌙"
  },
  "tools": { "profile": "full" }
}

OpenClaw comes with built-in skills you can enable during onboarding — favorites: agent-config and skill-creator. You can also create your own skills. Browse more on ClawHub 🏪

Skills are markdown files that teach your agent what to do. Agents are configured in JSON with their own model, personality, and tools. OpenClaw comes with built-in skills — agent-config and skill-creator are the best to start with.

💓 HEARTBEAT.md

Scheduled cron jobs — autonomous tasks

# Rafa’s Cron Jobs
08:00 daily briefing
*/30m health check
22:00 daily summary
on_error self-correct
Everything is markdown. SKILL defines abilities, SOUL defines personality, MEMORY persists context, IDENTITY is who the agent is, TOOLS for integrations, USER for your preferences, AGENTS for multi-agent setups, and HEARTBEAT for scheduled autonomous tasks like daily briefings.

This is the full architecture. On the left, users interact via any messaging channel. In the center is the self-hosted OpenClaw system running on your VPS — the gateway routes messages to your agent, which is powered by an external LLM API. The agent has its personality via SOUL.md, identity via IDENTITY.md, and orchestration via AGENTS.md. It uses modular skills to execute tasks, has persistent memory, and accesses local resources. The output — PRs, summaries, status updates — flows back to your channel.
